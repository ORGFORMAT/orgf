# Integrations

This directory is reserved for public tools that exchange hierarchical data between ORGF and external systems, file formats, or services.

An integration may:

- import external data into ORGF;
- export ORGF data to an external target;
- support a two-way workflow where the mapping and update rules are explicit.

Every integration must document its supported source and target versions, hierarchy and field mapping, treatment of incomplete or unsupported data, execution instructions, and licence.

No executable integration is included in ORGF 1.0.0.

## Planned direction

An early candidate is a converter between 1C staffing-schedule data and ORGF. Its scope, supported 1C export form, mapping rules, and licence will be published with the implementation rather than implied by this placeholder.
